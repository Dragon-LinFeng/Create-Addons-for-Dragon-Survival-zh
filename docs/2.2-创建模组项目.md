# 创建你的第一个龙之生存附属模组

## 📝 写在开始之前

这篇教程会带你创建一个完整的模组开发环境。如果你是新手，请按顺序一步一步来，不要跳过任何步骤。

---

## 第一步：确定你的龙种 ID

在项目创建之初，请务必为你的龙种确定一个英文 ID（例如，全小写加下划线）。这个 ID 将作为你整个模组的根基，后续会频繁使用。中途更改会非常麻烦。

> 💡 **命名规则**：全小写，单词间用下划线连接，例如我的附属「晶咒龙」，英文名是 `Crystcursed Dragon`，模组 ID 定为 `crystcursed_dragon`。

---

## 第二步：使用 IDEA 创建模组项目

### 2.1 新建项目

1. 打开 IDEA，点击「新建项目」
2. 左侧生成器栏选择 **Minecraft**

### 2.2 填写项目信息

这里以我的晶咒龙作为例子

| 配置项 | 示例 | 说明 |
|--------|------|------|
| **名称** | `Crystcursed-Dragon` | 项目显示名称 |
| **位置** | `D:/Mods/Crystcursed-Dragon` | 项目存放路径 |
| **创建 Git 仓库** | ✅ 勾选 | 便于版本管理 |
| **Groups** | `Mod` | 保持默认 |
| **Templates** | `NeoForge` | 必须与龙之生存一致 |
| **Language** | `Java` | 保持默认 |
| **使用 Mixins** | ❌ 不勾选 | 新手建议跳过 |
| **Mod ID** | `crystcursed_dragon` | 你刚才想好的ID |
| **Mod 名称** | `Crystcursed Dragon` | 显示给玩家看的名字 |
| **主类** | 自动生成 | 不用管 |
| **许可证** | `All Rights Reserved` | 新手保持默认 |

### 2.3 可选设置 (Optional Settings)

- **简介**：简单描述你的模组
- **作者**：你的公开ID（建议和CurseForge用户名一致）
- **网页/更新URL**：可留空

### 2.4 构建系统配置

| 配置项 | 示例 | 说明 |
|--------|------|------|
| **组 ID** | `by.linfeng` | 格式：`by.作者ID` |
| **工件 ID** | `crystcursed_dragon` | 建议和Mod ID一致 |
| **版本** | `0.1.0` | 从0.1.0开始，遵循[语义化版本控制](https://semver.org/lang/zh-CN/) |

### 2.5 JDK

选择 **Java 21**，如果没有，点击「从磁盘添加JDK...」手动添加。

确认无误后，点击「创建」。

> ⚠️ **构建失败** 在国内网络环境可能会出现构建失败的情况，在稍后会介绍如何处理，请先按照步骤继续配置。

## 第三步：配置 CurseMaven 仓库

CurseMaven 让我们能下载CurseForge上的模组作为依赖。

打开 `build.gradle`，在 `repositories` 块中添加：

```gradle
repositories {
    mavenLocal()  // 先找本地缓存
    maven {       // CurseMaven 仓库
        url "https://cursemaven.com"
        content {
            includeGroup "curse.maven"  // 只处理这个组的依赖
        }
    }
    // 其他仓库...
}
```

> 💡 **为什么要有 `content`？**  
> 告诉Gradle：只有 `curse.maven` 这个组的依赖才去CurseMaven找，其他依赖还是从默认仓库下载。

---

## 第四步：添加龙之生存作为依赖

### 4.1 获取依赖信息

1. 打开龙之生存的 [CurseForge页面](https://www.curseforge.com/minecraft/mc-mods/dragons-survival)
2. 找到你需要的版本（例如 [v2.0.52](https://www.curseforge.com/minecraft/mc-mods/dragons-survival/files/7320820)）
3. 在页面找到 **Curse Maven Snippet**，复制类似这样的内容：

   ```
   implementation "curse.maven:dragons-survival-420799:7320820"
   ```

### 4.2 添加依赖

打开 `build.gradle`，在 `dependencies` 块中添加：

```gradle
dependencies {
    // 龙之生存本体
    implementation "curse.maven:dragons-survival-420799:7320820"
    
    // GeckoLib（龙之生存的前置）
    implementation "curse.maven:geckolib-388172:6920810"
}
```

> ⚠️ **注意**：龙之生存依赖 GeckoLib，必须一并添加，否则会报错！

## 第五步：配置模组依赖关系

告诉NeoForge：我的模组需要龙之生存，且必须在它之后加载。

### 5.1 修改 build.gradle

找到 `generateModMetadata` 任务，添加版本范围变量：

```gradle
var generateModMetadata = tasks.register("generateModMetadata", ProcessResources) {
    var replaceProperties = [
            // ... 原有的属性保持不动
            "dragonsurvival_version_range": dragonsurvival_version_range  // 新增这行
    ]
    // ...
}
```

### 5.2 修改 neoforge.mods.toml

打开 `./src/main/templates/META-INF/neoforge.mods.toml`，在文件末尾添加：

```toml
# 依赖关系：龙之生存
[[dependencies.${mod_id}]]
modId = "dragonsurvival"
type = "required"
versionRange = "${dragonsurvival_version_range}"
ordering = "AFTER"
side = "BOTH"
```

### 5.3 修改 settings.gradle

打开 `./settings.gradle`，在文件末尾添加：

```gradle
// 龙之生存版本范围
// [2.0.52,) 表示需要 2.0.52 及以上版本
dragonsurvival_version_range = '[2.0.52,)'
```

## 第六步：配置国内镜像（解决网络问题）

初次构建可能因为网络问题失败，我们需要配置国内镜像加速。这里选用腾讯源，你也可以使用其他源作为镜像源。

### 6.1 修改 Gradle 下载源

打开 `./gradle/wrapper/gradle-wrapper.properties` 文件，将 `distributionUrl` 替换为：

```properties
distributionUrl=https\://mirrors.cloud.tencent.com/gradle/gradle-8.8-bin.zip
```

### 6.2 修改 build.gradle 文件

找到 `repositories` 块，添加腾讯镜像：

```gradle
repositories {
    maven {
        name = "TencentMirror"
        url = "https://mirrors.cloud.tencent.com/nexus/repository/maven-public/"
    }
    // 原有的其他仓库保持不动
}
```

### 6.3 修改 settings.gradle 文件

找到 `pluginManagement.repositories` 块，按照如下示例添加镜像源内容

```gradle
pluginManagement {
    repositories {
        maven {
            url = "https://mirrors.cloud.tencent.com/nexus/repository/maven-public/"
        }
        maven {
            url = "https://mirrors.cloud.tencent.com/nexus/repository/gradle-plugins/"
        }
        gradlePluginPortal()
        mavenCentral()
        // ... 其他原有配置
    }
}
```

### 6.4 刷新项目

展开IDEA右侧Gradle面板(小象图标)中的「刷新」按钮（🔄），等待同步完成。

你展开位于IDEA左侧的锤子图标后查看构建进度

## 第七步：配置数据包开发环境（VSCode）

这一步是为开发数据包内容准备的。

### 7.1 创建基础文件夹

在项目根目录下创建：

- `doc/` - 存放文档
- `object/` - 存放美术资源（模型、纹理源文件等）

### 7.2 配置 resources 文件夹

1. 在文件资源管理器中打开 `./src/main/resources`
2. 右键选择「通过 Code 打开」
3. 在VSCode中创建：
   - `data/` 文件夹
   - `pack.mcmeta` 文件

### 7.3 编写 pack.mcmeta

```json
{
    "pack": {
        "pack_format": 48,
        "description": ""
    }
}
```

### 7.4 配置 Spyglass 补全

在之前的 **基础环境部署** 教程中我们已经配置过 dragonsurvival-mcdoc-completion-zh 作为补全核心了

1. 将补全项目中的 `spyglass.json` 复制到 `./src/main/resources`
2. 按如下示例修改 `spyglass.json`（**注意替换为你的实际路径**）：

```json
{
    "env": {
        "useFilePolling": true,
        "dependencies": [
            "@vanilla-mcdoc",
            "file:///D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh"
        ],
        "mcmetaSummaryOverrides": {
            "registries": {
                "path": "file:///D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh/spyglass/registries.json",
                "replace": false
            },
            "commands": {
                "path": "file:///D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh/spyglass/commands.json",
                "replace": false
            }
        }
        // 其他配置
    }
}
```

### 7.5 创建 VSCode 工作区

1. **添加文件夹**：在 VSCode 的「文件」菜单中选择「将文件夹添加到工作区...」  
   - 添加 `doc/` 文件夹
   - 添加 `object/` 文件夹
2. **保存工作区**：在 VSCode 的「文件」菜单中选择「将工作区另存为...」  
   - 保存到 `./src/main/resources/`
   - 文件名：`你的模组ID.code-workspace`（如 `crystcursed_dragon.code-workspace`）
3. **添加到项目管理器**：打开侧边的「项目管理器」(文件夹图标)，点击「FAVORITES」区域的 💾 图标，将这个工作区保存下来，方便日后通过这里快速访问。

这样工作区配置就可以一并纳入 Git 管理了。

## 第八步：优化打包配置

防止开发配置文件被打包进最终的模组JAR文件。

1. 打开 `build.gradle`，修改 `sourceSets` 配置：

2. 注释掉原有的 `sourceSets.main.resources { srcDir 'src/generated/resources' }` 这一行。

3. 在其下方添加更精确的配置：

   ```gradle
   sourceSets {
       main {
           resources {
               srcDirs = ['src/main/resources', 'src/generated/resources']
               
               // 明确指定需要打包进 JAR 的文件夹和文件
               include 'assets/**'
               include 'data/**'
               include 'META-INF/**'   // 必须包含，否则模组无法加载
               include 'pack.png'
               include 'pack.mcmeta'
               
               // 所有其他文件（如 spyglass.json, *.code-workspace 等）都会被自动排除
           }
       }
   }
   ```
