# 创建你的第一个模组项目

## 1. 决定你的龙种 ID

在项目创建之初，请务必为你的龙种确定一个英文 ID（例如，全小写加下划线）。这个 ID 将作为你整个模组的根基，后续会频繁使用。中途更改会非常麻烦。

- **示例**：我的附属「晶咒龙」，英文名是 `Crystcursed Dragon`，模组 ID 定为 `crystcursed_dragon`。

## 2. 使用 IDEA 新建模组项目

1. 打开 IDEA，点击「新建项目」，在左侧生成器栏选择「Minecraft」。
2. 根据你的项目填写参数：
   - **名称**：你的项目名，例如 `Crystcursed-Dragon`。
   - **位置**：项目存放路径，例如 `D:/Programming/MinecraftMod/Dragonsurvival/Addon/`。
   - **创建 Git 仓库**：建议勾选，以便进行版本管理。
   - **Groups**：选择 `Mod`。
   - **Templates**：选择与龙之生存一致的 `NeoForge`。
   - **Language**：选择 `Java`。
   - **使用 Mixins**：如果你是新手，可以不勾选。
   - **Mod ID**：填入你刚决定的 ID，例如 `crystcursed_dragon`。
   - **Mod 名称**：建议与项目名称一致，例如 `Crystcursed Dragon`。
   - **主类**：自动生成，跳过。
   - **许可证**：如不了解，保持默认的 `All Rights Reserved`。
   - **可选设置 (Optional Settings)**：
     - **简介**：模组的简短描述。
     - **作者**：你的公开 ID，建议与发布平台的用户名一致。
     - **网页**：可留空，或等发布后再填写。
     - **更新 URL**：可留空。
   - **构建系统配置**：
     - **组 ID**：个人开发者通常使用 `by.<作者id>`，例如 `by.linfeng`。
     - **工件 ID**：建议与你的 Mod ID 一致，例如 `crystcursed_dragon`。
     - **版本**：建议从 `0.1.0` 开始，遵循[语义化版本控制](https://semver.org/lang/zh-CN/)。
   - **JDK**：选择你安装的 Java 21。如果列表中没有，可以点击「从磁盘添加 JDK...」手动添加。
3. 确认无误后，点击「创建」。

## 3. 解决模组构建失败（可选）

由于网络原因，初次构建可能会失败。可以通过替换为国内镜像来解决。

1. **修改 Gradle 下载源**：打开 `./gradle/wrapper/gradle-wrapper.properties`，将 `distributionUrl` 替换为：

   ```
   https\://mirrors.cloud.tencent.com/gradle/gradle-8.8-bin.zip
   ```

2. **添加 Maven 仓库镜像**：
   - 在 `./build.gradle` 的 `repositories` 块中，添加腾讯镜像源：

     ```java
     repositories {
         maven {
             url 'https://mirrors.cloud.tencent.com/nexus/repository/maven-public/'
         }
         mavenLocal()
         // ... 其他原有配置
     }
     ```

   - 在 `./settings.gradle` 的 `pluginManagement.repositories` 块中，添加腾讯镜像源：

     ```java
     pluginManagement {
         repositories {
             maven {
                 url 'https://mirrors.cloud.tencent.com/nexus/repository/maven-public/'
             }
             maven {
                 url 'https://mirrors.cloud.tencent.com/nexus/repository/gradle-plugins/'
             }
             mavenLocal()
             // ... 其他原有配置
         }
     }
     ```

3. 修改完成后，点击 IDEA 右侧 Gradle 面板的「刷新」按钮，重新同步项目。

## 4. 配置模组依赖关系

我们需要告诉 NeoForge，你的模组是龙之生存的附属，并且必须在它之后加载。

1. **修改 `build.gradle`**：找到 `generateModMetadata` 任务，在 `replaceProperties` 的 `Map` 中添加一行 `"dragonsurvival_version_range": dragonsurvival_version_range`。

   ```java
   var generateModMetadata = tasks.register("generateModMetadata", ProcessResources) {
       var replaceProperties = [
               // ... 其他原有属性
               "dragonsurvival_version_range": dragonsurvival_version_range // 添加此行
       ]
       // ...
   }
   ```

2. **修改 `neoforge.mods.toml`**：打开 `./src/main/templates/META-INF/neoforge.mods.toml` 文件，在文件末尾添加依赖关系。

   ```toml
   # 依赖关系：龙之生存
   [[dependencies.${mod_id}]]
   modId = "dragonsurvival"
   type = "required"
   versionRange = "${dragonsurvival_version_range}"
   ordering = "AFTER"
   side = "BOTH"
   ```

3. **修改 `settings.gradle`**：打开 `./settings.gradle` 文件，在文件末尾添加版本范围的定义。

   ```gradle
   // 龙之生存的版本范围
   // 注意：这里使用的是方括号 `[` 和 `)`，表示包含 2.0.52 及以上的所有版本
   dragonsurvival_version_range = '[2.0.52,)'
   ```

## 5. 整合 VSCode 与数据包开发环境

1. **创建必要的文件夹**：在项目根目录下创建 `doc/`（存放文档）和 `object/`（存放美术源文件等资源）。

2. **配置 `resources` 文件夹**：
   - 在文件资源管理器中找到项目的 `./src/main/resources` 文件夹。
   - 右键点击，选择「通过 Code 打开」。
   - 在打开的 VSCode 窗口中，创建 `data/` 文件夹和 `pack.mcmeta` 文件。
   - 向 `pack.mcmeta` 中写入以下内容，让 Spyglass 识别这是一个数据包：

     ```json
     {
         "pack": {
             "pack_format": 48,
             "description": ""
         }
     }
     ```

3. **链接补全核心**：
   - 将`D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh/`中的 `spyglass.json` 文件复制到项目的 `./src/main/resources` 文件夹中。
   - 修改以下配置（**请将路径替换为你自己的实际路径**），让 Spyglass 能够链接之前安装的补全项目：

     ```json
     {
         "env": {
             "useFilePolling": true,
             "dependencies": [
                 "@vanilla-mcdoc",
                 "file:///D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh"
             ],
             "mcmetaSummaryOverrides": {
                 "registries": {
                     "path": "file:///D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh/spyglass/registries.json",
                     "replace": false
                 },
                 "commands": {
                     "path": "file:///D:/Programming/McDoc/dragonsurvival-mcdoc-completion-zh/spyglass/commands.json",
                     "replace": false
                 }
             }
             // 其他配置
         }
     }
     ```

4. **创建 VSCode 工作区**：
   - 在 VSCode 的「文件」菜单中选择「将文件夹添加到工作区...」，分别将项目根目录下的 `doc/` 和 `object/` 文件夹添加进来。
   - 再次点击「文件」->「将工作区另存为...」，将工作区文件保存在 `./src/main/resources/` 目录下，并以你的 Mod ID 命名（例如 `crystcursed_dragon.code-workspace`）。这样工作区配置就可以纳入 Git 管理了。
   - 最后，打开「项目管理器」，点击「FAVORITES」区域的 💾 图标，将这个工作区也保存下来，方便日后快速访问。

## 6. 优化打包配置（添加白名单）

为了防止将开发环境中的配置文件（如 `spyglass.json`、`.code-workspace` 等）打包进最终的 `.jar` 文件，我们需要在 `build.gradle` 中明确指定要包含的文件。

1. 找到 `./build.gradle` 文件，注释掉原有的 `sourceSets.main.resources { srcDir 'src/generated/resources' }` 这一行。
2. 在其下方添加更精确的配置：

   ```java
   sourceSets {
       main {
           resources {
               srcDirs = ['src/main/resources', 'src/generated/resources']
               // 明确指定需要打包进 JAR 的文件夹和文件
               include 'assets/**'
               include 'data/**'
               include 'META-INF/**'   // 必须包含，否则模组无法加载
               include 'pack.png'
               include 'pack.mcmeta'
               // 所有其他文件（如 spyglass.json, *.code-workspace 等）都会被自动排除
           }
       }
   }
   ```

---

至此，你的模组项目已经创建并配置完成，可以正式开始创作了！
